# 工具：Cloudflare+GitHub+Hexo Fluid


## Hexo
hexo 是一个快速、简洁，而且功能强大的静态博客框架。我们可以使用 Markdown 编写博客文章，然后 hexo 帮我们把 Markdown 文件渲染成静态 HTML 页面。因此 hexo 非常适合用来搭建技术类博客，以及项目文档和个人网站。

需要安装node.js依赖

## github托管
将博客项目托管到github仓库，并使用Github pages来部署博客，可以让其他人也访问该博客。

## Cloudflare
使用cloudflare加速国内用户访问速度。

# 使用
## 发布新文章
### 方式一：发布语雀原生导出的 md
例如现在有一个语雀导出的文件 `bloga.md`，要作为 2026 年 5 月的文章发布。

1.将 `bloga.md` 放到对应月份目录：
```
source/_posts/2026/05/bloga.md
```

2.进入 Hexo 项目根目录，执行导入命令：
```
cd /path/to/my-blog
npm run import:yuque
```

这条命令会自动完成以下事情：
- 可以一次处理多篇md
- 将裸 md 转成文章包结构：`source/_posts/2026/05/bloga/bloga.md`
- 创建图片目录：`source/_posts/2026/05/bloga/img/`
- 下载语雀 CDN 图片到 `img/`
- 将 md 里的图片引用改成 `img/...`
- 自动补充头信息：`title`、`date`、`categories`、`tags`

3.导入后检查文章：
```
npm run verify:posts
```

如果校验通过，再打开：
```
source/_posts/2026/05/bloga/bloga.md
```

把 `categories: [xx]` 和 `tags: xxx` 改成实际分类和标签。

4.本地预览：
```
npm run clean
npm run build
npm run server
```

浏览器打开：
```
http://localhost:4000/
```

5.确认无误后发布：
```
npm run clean
npm run build
npm run deploy
```

### 方式二：手写新文章
在 Hexo 项目根目录下执行：
```
hexo new "Hello World"
```

该命令会生成：
```
source/_posts/YYYY/MM/Hello-World/Hello-World.md
```

每篇文章独立一个文件夹，文章图片放在同级 `img/` 目录，md 中统一使用 `img/xxx.png`。
