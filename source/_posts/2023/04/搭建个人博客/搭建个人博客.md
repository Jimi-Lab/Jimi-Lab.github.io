# 工具：Cloudflare+GitHub+Hexo Fluid


## Hexo
hexo 是一个快速、简洁，而且功能强大的静态博客框架。我们可以使用 Markdown 编写博客文章，然后 hexo 帮我们把 Markdown 文件渲染成静态 HTML 页面。因此 hexo 非常适合用来搭建技术类博客，以及项目文档和个人网站。

需要安装node.js依赖

## github托管
将博客项目托管到github仓库，并使用Github pages来部署博客，可以让其他人也访问该博客。

## 多端同步发表blog

### 从source下载源码

在新的电脑上维护博客时，只需要下载 `source` 分支，不需要手动下载 `main` 分支。`source` 分支保存 Hexo 全部源码，`main` 分支由 `hexo deploy` 自动更新，用来展示最终网页。

第一次在新电脑上执行：
```
git clone -b source git@github.com:Jimi-Lab/Jimi-Lab.github.io.git my-blog
cd my-blog
npm install
```

如果是在 Windows 上使用 Git，建议额外开启长路径支持：
```
git config --global core.longpaths true
git config core.longpaths true
```

以后每次开始写文章前，先同步最新源码：
```
cd /path/to/my-blog
git checkout source
git pull
```




### 添加blogs

以语雀原生导出的 `bloga.md` 为例，如果这篇文章属于 2026 年 5 月，就先把文件放到：
```
source/_posts/2026/05/bloga.md
```

然后在 Hexo 项目根目录执行：
```
cd /path/to/my-blog
npm run import:yuque
npm run clean:yuque-style
npm run verify:posts
```

这几条命令会完成：
- 可以一次处理多篇md
- 将裸 md 转成文章包结构：`source/_posts/2026/05/bloga/bloga.md`
- 将远程图片下载到同级 `img/` 目录
- 将 md 中的图片引用统一改成 `img/...`
- 清理语雀导出的 `<font style="...">`、空 `<font></font>` 等低价值样式标签
- 对原本带背景色高亮的内容自动保留为加粗文本
- 检查是否还有远程图片、旧图片路径、缺失图片、裸 md 等问题

校验通过后，打开生成后的文章 md，手动修改分类和标签：
```
source/_posts/2026/05/bloga/bloga.md
```

需要重点修改：
```
categories: [xx]
tags:
  - xxx
```




### Push至github source branch

文章导入和分类标签确认后，先确认自己在 `source` 分支，再把源码同步到 GitHub：
```
git status
git checkout source
git pull --rebase origin source
git add -A
git commit -m "update blog source"
git push
```

如果 `git status` 显示当前分支不是 `source`，必须先切到 `source`。日常维护源码不要在 `main` 分支提交，`main` 分支只交给 `npm run deploy:site` 发布网页。


本地预览：
```
npm run preview:site
```


生成并发布网页：
```
npm run deploy:site
```

这里需要区分：
- `git push`：保存源码到 `source` 分支，方便 Windows / Mac / Linux 多端同步
- `npm run deploy:site`：依次执行 `clean`、`build`、写入 `public/README.md`、`deploy`，把 Hexo 生成后的静态网页发布到 `main` 分支
